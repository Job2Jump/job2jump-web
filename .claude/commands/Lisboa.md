Guarda todos los cambios del proyecto y los sube a GitHub para que el otro pueda descargarlos.

Ejecuta en orden:
1. `git add .` — prepara todos los archivos modificados
2. `git commit -m "Lisboa - $ARGUMENTS"` — si el usuario ha escrito algo después de /Lisboa úsalo como mensaje, si no usa "Lisboa - actualización"
3. `git push` — sube los cambios a GitHub

Cuando termine, confirma que los cambios están subidos y que el otro puede ejecutar /blacky para tenerlos.
